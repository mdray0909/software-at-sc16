Automatic Test Generation

  • Towards Automatic and Flexible Unit Test Generation for Legacy HPC Code (Hovy and Kunkel)
  • FortranTestGenerator tool
  • Capture and Replay
    • Extracts data while running original application
    • Static code analysis to reduce captured variables
    • Generate unit tests based on data and templates
  • Finds the places most likely to be affected by change, and creates tests for them
  • Ongoing work to implement "mocking" for MPI

Software Metrics

  • Development Effort Estimation in HPC (Wienke et al)
  • tl;dr: existing industry models don't apply to HPC
    • Standard complexity metrics give widely varied results because of widespread use of directives, optimising flags, process binding etc.
    • Standard cost estimation models (e.g. COCOMO II) don't work because of industry biases: effort split, team size, ranking of software attributes
    • Wienke defines revised cost model for HPC focussed on a performance life-cycle
